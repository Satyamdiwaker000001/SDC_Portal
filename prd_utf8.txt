Part 1
Understanding SDC
What is the Software Development Cell?
The Software Development Cell (SDC) is a student-run technical organization within the college that gives
students a real-world platform to sharpen their development skills, collaborate on meaningful projects, and
grow into industry-ready professionals ΓÇö all while still in college.
Our Mission: To bridge the gap between academic learning and professional software engineering by
giving students hands-on project experience, senior mentorship, and a culture of continuous growth.

Seniors guide juniors through technical challenges, code reviews, and career
decisions.
Industry Practices
Learn Git, pull requests, code reviews, agile sprints, and professional

Part 2
Why Are We Building the SDC Website?
The Problem Today
Currently SDC has no central platform. Projects are tracked manually, members have no public presence,
recruitment happens informally, and there is no way for the outside world ΓÇö or new students ΓÇö to
discover what SDC has built.
Current Problem
Impact
No central project tracking
Projects are scattered across chats and spreadsheets with no single source
of truth.
No member visibility
Talented SDC members have no unified showcase for their work and
contributions.
No formal recruitment process
New student intake is informal and inconsistent.
No admin control panel
Admins cannot efficiently manage teams, tasks, or announcements.
No transparency for juniors
New members can't see ongoing projects, team structures, or their assigned
tasks.
What the SDC Website Will Solve
What We Build
How It Helps
Web & App Project Showcase
A public landing page displaying all SDC projects ΓÇö live, completed, and
in-progress.
Team Collaboration Portal
Internal dashboard for Admins to create teams, assign projects, and monitor
progress.
Task Management System
Developers can view assigned tasks, update progress, and submit for review.
Workshops & Coding Events
Announcements section to promote upcoming workshops and coding
sessions.
Senior Guidance Visibility
Profiles of seniors and alumni, linked to their GitHub and LinkedIn for
mentorship.
Recruitment Form
A public application form for new students to apply to join SDC.
Announcements System

Part 7
System Architecture
The SDC Portal follows a clean client-server architecture with a clear separation of concerns across three
layers.
1. Presentation Layer
(Frontend)
React + TypeScript
Tailwind CSS
Public Landing Page (Next.js/React) Admin Dashboard
(React) Developer Portal (React) All pages communicate
with backend via REST API calls
2. Application Layer
(Backend)
FastAPI (Python)
JWT Auth
SQLAlchemy ORM
Handles all business logic Enforces role-based access
control Processes requests from frontend Returns
structured JSON responses Auto-generates Swagger API
docs at /docs
3. Data Layer
(Database)
MySQL Normalized
Schema
Stores all persistent data 9 relational tables with FK
constraints Foreign keys ensure data integrity JSON
columns used for flexible fields (techStack)
Request Flow
Step
What Happens
Step 1
User opens SDC Portal (browser / mobile)
Step 2
React app sends HTTP request to FastAPI backend
Step 3
FastAPI validates JWT token and checks role permissions
Step 4
Backend queries MySQL database via SQLAlchemy ORM
Step 5
Database returns data to backend
Step 6
FastAPI formats and returns JSON response
Step 7
React renders the data to the user
